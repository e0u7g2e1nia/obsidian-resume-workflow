# 简历命名与投递记录规范

> 用途：记录简历输出、投递记录、Dataview 汇总的命名与字段规则。之后如果工作流调整，优先修改本文件。

---

## 1. 文件命名规则

统一使用：

```text
YYYY-MM-DD｜类型｜公司｜岗位｜可选版本.md
```

字段说明：

- `YYYY-MM-DD`：生成日期 / 投递记录创建日期。
- `类型`：`投递记录` 或 `简历输出`。
- `公司`：公司名。
- `岗位`：岗位名。
- `可选版本`：仅在需要区分版本时使用，如 `v1`、`终版`、`面试版`。

### 示例

```text
2026-06-09｜投递记录｜爱奇艺｜AI NPC项目内容运营实习生.md
2026-06-09｜简历输出｜爱奇艺｜AI NPC项目内容运营实习生.md
2026-06-09｜简历输出｜爱奇艺｜AI NPC项目内容运营实习生｜终版.md
```

## 3. 投递记录 YAML

每个岗位一篇投递记录，用于存放 JD 原文、投递状态、简历链接和后续进展。

```yaml
---
type: 投递记录
company: 爱奇艺
position: AI NPC项目内容运营实习生
status: 已生成简历
direction: AI内容运营
apply_date: 2026-06-09
priority: 高
resume: "[[01实习/简历输出/2026-06-09｜简历输出｜爱奇艺｜AI NPC项目内容运营实习生]]"
source:
---
```

建议正文结构：

```md
# 爱奇艺｜AI NPC项目内容运营实习生

## 原始JD

## 投递记录

## 简历策略

## 后续进展
```

## 4. 简历输出 YAML

每份定制简历也可以带 YAML，方便回溯对应岗位记录。

```yaml
---
type: 简历输出
company: 爱奇艺
position: AI NPC项目内容运营实习生
direction: AI内容运营
created_date: 2026-06-09
record: "[[01实习/投递记录/2026-06-09｜投递记录｜爱奇艺｜AI NPC项目内容运营实习生]]"
status: 可投递
---
```

---

## 5. Dataview 汇总表

`[[01实习/JD投递汇总.md]]` 可使用：

```dataview
TABLE
  company AS 公司,
  position AS 岗位,
  status AS 状态,
  direction AS 方向,
  apply_date AS 日期,
  priority AS 优先级,
  resume AS 简历
FROM "01实习/投递记录"
WHERE type = "投递记录"
SORT apply_date DESC
```

---

## 6. 当前约定

- Dataview 不单独存数据，只从每篇 `投递记录` 的 YAML 中读取信息。
- 一个岗位对应一篇 `投递记录`。
- 一份定制简历对应一篇 `简历输出`。
- 文件名给人眼识别，YAML 给 Dataview 查询。
- 后续若规则变化，优先修改本规范笔记，再同步已有文件。
